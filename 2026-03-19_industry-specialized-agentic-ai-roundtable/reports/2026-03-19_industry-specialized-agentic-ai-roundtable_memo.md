# Memo

## One-Line Read
이 간담회가 말하는 핵심은 한국이 파운데이션 모델 경쟁을 그대로 따라가기보다 `산업별 workflow와 데이터 경계를 잘 아는 Agentic AI 운영층`에서 승부해야 한다는 점이다.

## 핵심 요약
- 조동근은 agent를 `LLM + tool use + memory + feedback loop`로 설명하면서 `MCP`, `Skills`, `A2A` 같은 표준이 실전 경쟁력을 좌우한다고 본다.
- 김성훈은 지금 agent가 “버즈워드만은 아닌 시점”이라고 보지만, 실제 산업 도입의 핵심 병목은 `문서`, `설명 가능성`, `프로세스 데이터`, `현업 참여`라고 본다.
- 질의응답의 중심은 기술 쇼케이스가 아니라 `어디에 먼저 적용할 것인가`, `지금 투자해야 하는가`, `보안과 국내 공급망을 어떻게 볼 것인가`, `공공과 민간의 데이터 역할을 어떻게 나눌 것인가`였다.
- 영상 전체의 결론은 범용 agent 환상보다 `버티컬 프로세스 자동화`, `내부 업무 AX`, `공공 인허가`, `문서 중심 심사`, `보안이 중요한 산업`이 더 현실적이라는 데 가깝다.
- 이 방향은 외부 공식 자료와도 맞는다. Anthropic은 `단일 agent + domain skills`로 시작하라고 하고, UiPath와 Salesforce도 기존 workflow와 신뢰 계층을 중심으로 enterprise agent를 제품화하고 있다.

## 총체적 리뷰
- 장점: 현장의 문제의식이 명확하다. 특히 `프로세스 데이터가 없다`, `현업 전문가가 필요하다`, `보안과 수출 전략을 같이 봐야 한다`는 지점은 현실적이다.
- 아쉬움: agent 평가 체계, 실패율 관리, human-in-the-loop 설계, 운영 관측성에 대한 논의는 상대적으로 얕다.
- 의미: 그럼에도 이 간담회는 한국형 Agentic AI 전략이 어디서 시작돼야 하는지 꽤 정확하게 짚는다. 출발점은 범용 consumer agent가 아니라 `산업별 고통이 큰 업무 흐름`이다.

## 바로 적용할 인사이트
- 산업 특화 Agentic AI는 `도메인 UI`보다 `도메인 workflow`가 먼저다.
- 첫 제품은 “다 해주는 agent”보다 `문서 판독 + 규칙 체크 + 근거 제시 + 승인 요청`처럼 좁고 비싼 업무에서 시작하는 편이 맞다.
- 한국 시장에서는 `보안`, `국내 배치`, `민감 데이터 통제`, `공공 실증`, `수출 가능 표준화`를 같이 설계해야 한다.
- 조직 관점에서는 코딩 교육만 확대하는 것으로 부족하다. `AX 전문가`, `프로세스 설계자`, `도메인 평가자`를 함께 키워야 한다.

## NotebookLM Cross-Check
- 동일 소스를 NotebookLM에 다시 넣어 질의한 결과도 같은 결론으로 수렴했다. 핵심 원칙은 `workflow operating layer 선점`, `공공 reference workflow의 실증과 수출화`, `보안·감사 가능성을 포함한 trust layer 구축`이었다.
- 반대로 피해야 할 방향도 분명했다. `전사적 거대 플랫폼 선행 투자`, `UI만 산업형으로 포장한 가짜 버티컬`, `human escalation 없이 밀어붙이는 맹목적 자동화`는 모두 실패 확률이 높다.
- 따라서 경영진 관점의 실행 문장은 다음처럼 정리할 수 있다. `크게 사지 말고 좁게 시작하되, workflow 자산과 trust layer는 반드시 남겨라.`

## 권고
1. 산업별 PoC가 아니라 `업무 단위 reference workflow`부터 만들어야 한다.
2. 문서 중심 산업부터 공략하고, 이후 화면 조작형 agent로 확장하는 순서가 안전하다.
3. 공공은 일반화 가능한 프로세스 데이터와 평가셋을 만들고, 민간은 그 위에 버티컬 제품을 얹는 구조가 적절하다.
4. Agent 전략의 KPI는 데모 수가 아니라 `처리시간 단축`, `재작업률 감소`, `승인 대기시간 감소`, `보안 사고 없음`이어야 한다.
