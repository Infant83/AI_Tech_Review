Korea Does Not Need To Win The Model Race To Win The Workflow Race.

국가인공지능전략위원회의 2026년 3월 17일 간담회는 Agentic AI를 “범용 AI 쇼케이스”가 아니라 산업별 workflow, 문서, 승인 절차, 보안 경계를 재설계하는 문제로 다뤘다. 핵심 결론은 단순하다. 산업 특화는 모델 튜닝보다 도메인 스킬과 프로세스 자산의 축적 경쟁이다.

Most Defensible Insight

Workflow Asset > Model Hype

이 간담회가 가장 정확하게 본 것은 시장의 가치가 foundation model 자체보다 산업별 절차, 문서 구조, 권한 모델, human approval 설계에 쌓인다는 점이다.

What Matters Now

  1. 단일 agent에 도메인 스킬을 붙여 좁은 업무부터 시작할 것
  2. 공공 인허가처럼 데이터화 가치가 큰 프로세스를 실증장으로 삼을 것
  3. 보안·국내 배치·감사 로그를 제품의 후순위가 아니라 핵심 가치로 둘 것

What Holds Up

영상의 핵심 논지는 외부 공식 자료와 잘 맞는다. Anthropic은 domain-specific skills를 갖춘 single-agent 접근을 권장하고, UiPath와 Salesforce는 enterprise agent를 기존 workflow와 trust layer 위에서 제품화하고 있다.

What Is Underspecified

평가 harness, 실패 감지, 관측성, 승인 규칙, rollback 전략은 영상보다 훨씬 더 중요하다. 산업 특화 Agentic AI는 데모보다 운영 품질 관리 체계가 먼저다.

Why This Matters

한국이 지금 쌓아야 할 자산은 범용 모델의 파라미터가 아니라 workflow asset, domain skill catalog, security-ready deployment pattern이다.

Industry-Specific Approach

산업 특화 Agentic AI는 산업 라벨을 붙인 범용 챗봇이 아니다. 아래 다섯 층이 함께 설계돼야 한다.

Layer 1

Domain Knowledge

용어집, 규정, 표준 서식, 예외 규칙, 판단 기준을 구조화한다.

Layer 2

Workflow Graph

단계, 입력 조건, 출력 조건, 승인 지점, 반려 사유를 명시한다.

Layer 3

Trusted Tools

문서 파서, OCR, 사내 API, 검색, 결재 연동을 권한 단위로 묶는다.

Layer 4

Trust Layer

근거 표시, 감사 로그, 재현성, 데이터 반출 통제, human handoff를 넣는다.

Layer 5

Evaluation

처리시간, 재작업률, 승인 전환율, 오류 유형, 근거 적절성으로 측정한다.

Priority

Start Narrow

처음부터 범용 OS를 만들지 말고, 문서·심사·인허가·내부 업무처럼 좁고 비싼 작업부터 시작한다.

Best First Verticals

Public Permits

서류 누락 확인, 사전 적합성 검토, 보완 요청, 승인 전 정리까지 agent가 담당하기 좋다.

Manufacturing Quality Docs

도면, 스펙, 검사 기준, 불량 사유를 연결해 근거 기반 판정을 만드는 데 적합하다.

Finance / Insurance Review

문서 중심, 규칙 중심, audit trail 요구가 강해 산업 특화 agent의 정석적인 출발점이 된다.

Export / Compliance Ops

규정, 현지화, 고객사 요구사항 정리를 자동화해 한국형 B2B product로 패키지화하기 좋다.

Investment Logic

Wrong Bet

전사 차원의 거대한 agent platform을 먼저 사고, 이후 use case를 찾는 방식.

Right Bet

8-12주 단위의 좁은 workflow pilot를 반복하며, skill과 workflow asset은 축적하고 모델은 교체 가능하게 두는 방식.

Key KPI

데모 수가 아니라 처리시간 단축, 재작업률 감소, 근거 품질, 승인 대기시간 감소, 보안 사고 없음.

Current Recommendation

“지금 크게 투자”가 아니라 “지금 작게 시작하되 workflow 자산은 반드시 남기는 투자”가 맞다.

NotebookLM Cross-Check

동일한 소스를 NotebookLM에 다시 넣어 질의한 결과도 같은 구조로 수렴했다. 이것은 외부 독립 검증은 아니지만, 적어도 이 페이지의 핵심 논지가 과도하게 한쪽으로 치우치지 않았는지 보는 2차 synthesis로는 유효했다.

Principle 1

모델 추격전보다 workflow operating layer의 표준화와 재사용성을 먼저 잡아야 한다.

Principle 2

공공 인허가를 단순 효율화가 아니라 수출 가능한 reference workflow의 실증장으로 써야 한다.

Principle 3

Trust layer를 옵션이 아니라 제품 핵심으로 설계해야 한국 시장에서 방어력이 생긴다.

Do Not 1

전사적 거대 multi-agent 플랫폼을 먼저 사고 나중에 use case를 찾는 순서로 가지 말아야 한다.

Do Not 2

UI와 프롬프트만 바꾼 가짜 버티컬로 만족하지 말고 workflow graph와 권한 모델까지 묶어야 한다.

Do Not 3

human escalation, process data, rollback 규칙 없이 자동화부터 밀어붙이면 pilot가 멈출 가능성이 높다.

Ralph Loop Check

Round 1 Findings

이 페이지는 영상의 메시지를 단순 요약이 아니라 정책·산업·보안·제품 설계 관점으로 재배치했다. 다만 실제 배포 전에는 mobile layout, citation density, local resource links를 브라우저에서 재검증해야 한다.

Round 3 Fixes

Playwright CLI 기반 데스크톱·모바일 전체 페이지 캡처를 수행했고, 모바일 히어로 타이포를 한 단계 낮춰 가독성을 개선했다. 이후 NotebookLM 결과를 본문에 합쳤고, export 자동화는 활성 Chrome 세션 충돌 때문에 후속 과제로 남겨 두었다.
