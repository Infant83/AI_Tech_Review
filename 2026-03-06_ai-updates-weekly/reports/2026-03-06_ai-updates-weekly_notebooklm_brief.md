# NotebookLM Brief

## Topic
2026년 3월 첫째 주 AI 업데이트를 통해 본 `AI agent operating layer`의 부상

## Audience
- AI 기술 동향을 빠르게 파악해야 하는 개발자
- 에이전트 제품 전략을 고민하는 PM과 리더
- 실무 적용 포인트를 찾는 엔지니어링 팀

## Core Thesis
이번 주 업데이트의 핵심은 개별 모델 출시보다도, `skills + memory + remote control + orchestration + approvals`를 중심으로 한 범용 에이전트 운영층이 새로운 제품 기준으로 부상하고 있다는 점이다.

## Five Story Beats
1. 모델 뉴스의 홍수 속에서도 진짜 변화는 앱이 아니라 에이전트 운영층에서 일어나고 있다.
2. GPT-5.3 Instant, GPT-5.4, Gemini 3.1, Mercury 2, LFM2 같은 릴리스는 속도, 비용, 컨텍스트, 툴 사용성 경쟁을 강화한다.
3. 실제 성능은 모델보다 하네스, 메모리, 권한, 실패 복구, 모델 라우팅에서 갈린다.
4. AT&T 사례는 멀티 에이전트 오케스트레이션이 비용 절감과 생산성 향상에 실질적으로 작동함을 보여준다.
5. Anthropic와 미 국방부 갈등은 안전성, 거버넌스, 조달 계약이 AI 제품 전략의 핵심 변수임을 보여준다.

## What The Slides Should Emphasize
- "모델 중심 경쟁"에서 "운영층 중심 경쟁"으로 이동하는 흐름
- 데스크톱 에이전트와 코딩 에이전트가 하나의 계열로 합쳐지는 장면
- 하네스 엔지니어링과 멀티 에이전트 라우팅이 실무 생산성의 핵심이라는 점
- 에이전트 시스템 설계에서 메모리, 권한, 승인 UX가 왜 중요한지
- 앞으로 SaaS가 사라진다기보다, SaaS가 에이전트의 실행 레이어로 재배치된다는 해석

## What The Infographic Should Emphasize
- 하나의 중심 도식:
  `User -> Agent -> Skills -> Memory -> Tools/Apps -> Approvals -> Outcome`
- 좌측에는 시장 변화:
  `Standalone SaaS -> Generic Agent + Skills`
- 중앙에는 기술 변화:
  `Model IQ -> Harness / Routing / Tooling`
- 우측에는 조직 변화:
  `Prompting -> Agent Onboarding / Operations / Governance`
- 하단에는 리스크:
  `Security`, `Memory Drift`, `Over-permissioning`, `Cost Explosion`, `Policy Pressure`
