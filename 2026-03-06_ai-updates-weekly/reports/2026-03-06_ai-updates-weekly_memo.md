# Memo

## One-Line Read
2026년 3월 첫째 주 AI 뉴스의 핵심은 "더 좋은 모델" 자체보다 "에이전트를 실제로 일하게 만드는 운영층"이 경쟁의 중심으로 이동하고 있다는 점이다.

## 핵심 요약
- SaaS 대체 서사는 단순 과장이 아니라, 이미 `skills + memory + scheduling + remote control + tool protocol` 조합으로 제품화되기 시작했다.
- 모델 경쟁은 계속되지만 제품 차별화는 점점 `harness`, 라우팅, 권한 통제, 메모리 구조, 작업 오케스트레이션으로 이동한다.
- OpenAI GPT-5.3 Instant, GPT-5.4, Gemini 3.1, Mercury 2 같은 릴리스는 "정확도"보다도 속도, 비용, 컨텍스트, 툴 사용성, 실시간 에이전트 적합성을 밀고 있다.
- CoPaw, OpenClaw 계열, Accomplish, CodeBuff 같은 도구군은 하나의 경향을 보여준다. 사용자는 여러 개의 전용 앱 대신 범용 에이전트에 스킬을 붙이는 방향으로 이동하고 있다.
- 엔터프라이즈 측면에서는 AT&T 사례가 중요하다. 비싼 단일 모델을 모든 곳에 쓰기보다, 상위 오케스트레이터와 저렴한 워커 모델을 조합하는 쪽이 비용과 지연시간 모두에서 유리하다는 신호다.
- 안전성 측면에서는 Anthropic와 미 국방부 갈등이 상징적이다. 자발적 안전 서약만으로는 경쟁 압력과 국가 권력 앞에서 지속되기 어렵다.
- 실무자에게 필요한 것은 "최고 모델 찾기"보다 "신뢰 가능한 에이전트 작업 환경 설계"다.

## 왜 중요한가
이 변화는 AI가 기존 소프트웨어 위에 덧붙는 보조 기능에서 멈추지 않는다는 뜻이다. 앞으로의 경쟁력은 기능 하나를 잘 만드는 SaaS보다, 데이터 접근권과 작업흐름을 안전하게 감싸고 실제 업무를 끝까지 수행하게 하는 에이전트 시스템 설계에서 나온다.

## 실무 관점 액션
- 도메인별로 작게 쪼갠 `skill`과 `workflow module`을 먼저 정의한다.
- 메모리는 하나의 거대한 파일이 아니라 `logs / curated memory / searchable memory`의 다층 구조로 설계한다.
- 모든 에이전트 도구 실행에는 권한 경계와 승인 흐름을 둔다.
- 단일 거대 모델에 올인하지 말고, 상위 계획 모델과 저비용 실행 모델을 분리한다.
- 평가 기준을 데모 품질이 아니라 실제 업무 완료율, 실패 복구율, 총 토큰 비용, human handoff 비율로 둔다.

## 추천 후속 작업
- 사내 혹은 개인 워크플로 한 개를 골라 "범용 에이전트 + 스킬" 구조로 다시 설계해 본다.
- NotebookLM용 발표 자료는 "모델 뉴스 요약"보다 "왜 운영층이 새 OS가 되는가"에 맞춰 스토리라인을 잡는다.

## 워크스페이스 운영 메모
- `AI_Tech_Review/`를 이 보고서 작업들의 기본 상위 폴더로 본다.
- 공용 도구는 루트의 `scripts/`와 `.automation/`에만 둔다.
- 새로운 리뷰를 루트에서 시작하면 토픽 폴더를 하나 만들고 그 아래에서 `notes / reports / notebooklm_exports`를 생성한다.
- 이미 토픽 폴더 안에서 시작한 작업이면 그 폴더 아래에 바로 생성하고, 다시 한 번 중첩 폴더를 만들지 않는다.
- `.automation/`은 계속 유지하되, NotebookLM용 임시 Chrome 프로필 복사본은 시스템 임시 디렉터리에서 만들고 작업 후 삭제한다.
- NotebookLM Playwright 스크립트는 이제 특정 주제명을 하드코딩하지 않고 현재 토픽 폴더 또는 명시한 토픽 경로를 기준으로 동작한다.
- 첫 예제 패키지는 중간 컨테이너 없이 `AI_Tech_Review/2026-03-06_ai-updates-weekly/`로 평탄화했다.
